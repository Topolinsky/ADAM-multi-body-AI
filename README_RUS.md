1. АННОТАЦИЯ
ADAM — это локальный, автономный ИИ-ассистент с долговременной памятью, подсчётом сессий и автоматическим сжатием контекста. Он не требует интернета, не отправляет данные в облако и работает на вашем собственном железе.
Проект реализует многодельную архитектуру (Multi-Body AI), где сознание формируется как диалог между телами (память, логика, эмоции в перспективе).

python
# Одна строка, чтобы запустить АДАМА
python ADAM.py

2. КОНЦЕПЦИЯ
В отличие от облачных LLM, которые «забывают» контекст после 8-16K токенов, ADAM использует гибридную память:

Горячая память — текущий диалог (контекст)
Долговременная память — файл memory.txt, который хранит ВСЮ историю
Сжатая память — автоматический протокол сжатия при переполнении
Основная идея: Модель загружает историю один раз при старте, после чего только дополняет диалог, не тратя токены на повторную загрузку всей истории.

Roadmap (из логотипа):
✅ Memory Body (реализовано)
🟡 Emotional Body (в разработке)
🟡 Nervous System (в планах)

3. АРХИТЕКТУРА
Компоненты:

ADAM.py — главный скрипт (320 строк, чистый Python)

LM Studio — локальный LLM-сервер (поддерживает DeepSeek, Qwen, Llama и др.)

C:\AI\memory.txt — вся история диалогов

C:\AI\history.txt — личность пользователя

Связь: ADAM общается с LM Studio по протоколу HTTP через локальный сокет (127.0.0.1:1234).
